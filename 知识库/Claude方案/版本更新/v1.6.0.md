# 知识库 v1.6.0 发布记录

## 发布日期

2026-07-15

## 版本概述

混合语义检索版本。从纯 grep 关键词匹配升级为「关键词 + 向量语义」混合检索引擎，解决"换了个说法就搜不到"的核心痛点。

## 核心新增

- `scripts/kb_search.py` (1235 行)：本地混合检索引擎
  - 词法检索：aliases > keywords > tags > 标题/正文，保持与旧版一致的优先级
  - 语义检索：intfloat/multilingual-e5-small 本地 Embedding 模型 (384 维向量)
  - SQLite + NumPy 存储，零外部依赖，100% 本地运行
  - SHA-256 增量索引：搜索前自动检测新增/修改/删除文件
  - 自动降级：模型不可用时回退到纯词法检索
- `tests/test_search_hybrid.py`：54 个单元测试（全部通过）
- `skills/search/SKILL.md`：更新搜索 Skill（优先语义检索 → 失败回退 grep）

## 技术选型

| 组件 | 选择 | 原因 |
|------|------|------|
| Embedding 模型 | intfloat/multilingual-e5-small | 中英文混合支持，CPU 可运行，476MB |
| 向量存储 | SQLite + NumPy BLOB | 131 篇规模足够，零运维 |
| 索引策略 | 搜索时增量索引 (SHA-256) | 覆盖手动编辑、Git 拉取、文件删除 |
| 降级策略 | embedding 不可用 → 纯词法 | 不阻塞搜索 |

## 验收结果

- 搜索"恢复上次中断的会话"能召回 checkpoint 相关文档（语义分数 0.89）
- aliases/keywords/tags 的排序优先级不退化
- 正文命中排在纯语义命中之前
- 同一文档的词法和语义结果合并去重
- 新增笔记自动索引，无需手动操作
- embedding 不可用时自动退回到 grep 流程
- 索引文件存在 ~/Library/Caches 下，不进 Git/Obsidian Sync
- 不发送任何笔记内容到外部服务

## 影响范围

- 搜索 Skill (`/search`) 行为变更：自动调用 kb_search.py，首次需等待 2-3 秒加载模型
- 新增依赖：sentence-transformers (pip3 install sentence-transformers)
- 新增磁盘占用：模型 ~476MB，索引 ~3MB
